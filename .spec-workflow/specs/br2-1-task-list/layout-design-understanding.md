# 任務列表頁面布局設計 - 我的理解

根據您修改後的 `layout-design.md`，我對新布局設計的理解如下：

## 關鍵變更理解

### 1. 第一行布局變更（最重要）

**原設計：**
- 第一行：所有篩選項目 + 右側操作按鈕 + 統計摘要（都在同一行）

**您修改後的設計：**
- **第一行分成兩行顯示**：
  - **第一行的第一行**：`[搜尋框] [開始月份] [結束月份] [負責人] [標籤] [狀態] [到期狀態] [隱藏已完成✓]`
  - **第一行的第二行**：`[服務類型] [標籤多選] [是否可開始] [我的任務] [刷新] [批量分配]`

**我的理解：**
- 第一行被拆分為兩個子行，避免所有項目擠在一行
- 基本篩選項目（搜尋、月份、負責人、標籤、狀態、到期狀態、隱藏已完成）放在第一子行
- 高級篩選項目（服務類型、標籤多選、是否可開始、我的任務）和操作按鈕（刷新、批量分配）放在第二子行
- **統計摘要從第一行移除**

### 2. 第二行布局變更

**原設計：**
- 第二行：`[全選客戶] [一鍵展開]`

**您修改後的設計：**
- 第二行：`[全選客戶] [一鍵展開] [33總任務] [33進行中] [0已完成] [14逾期] [33可開始]`

**我的理解：**
- 統計摘要**移到第二行**，與客戶選擇控制（全選客戶、一鍵展開）放在同一行
- 第二行包含：左側是客戶選擇控制，右側是統計摘要
- 這樣可以充分利用第二行的空間，避免統計摘要佔用第一行的寶貴空間

## 完整布局結構理解

```
FilterCard
├─ 第一行（實際是兩個子行）
│  ├─ 第一子行：[搜尋框] [開始月份] [結束月份] [負責人] [標籤] [狀態] [到期狀態] [隱藏已完成✓]
│  └─ 第二子行：[服務類型] [標籤多選] [是否可開始] [我的任務] [刷新] [批量分配]
│
└─ 第二行：[全選客戶] [一鍵展開] | [33總任務] [33進行中] [0已完成] [14逾期] [33可開始]
```

## 設計優勢理解

1. **避免第一行過於擁擠** - 將篩選項目分成兩行，每行項目數量更合理
2. **統計摘要獨立放置** - 移到第二行，不與主要篩選項目競爭空間
3. **充分利用第二行空間** - 第二行同時放置客戶選擇控制和統計摘要，空間利用更高效
4. **邏輯分組更清晰** - 基本篩選 vs 高級篩選+操作按鈕，分組更明確

## 需要確認的問題

1. **第一行的兩個子行是否需要視覺分隔？** 
   - 是否需要 border 或背景色區分？
   - 還是僅靠間距自然分隔？

2. **第二行的布局方式？**
   - 是 `[全選客戶] [一鍵展開]` 在左側，統計摘要在右側（justify-content: space-between）？
   - 還是所有項目連續排列（gap: 8px）？

3. **響應式行為？**
   - 第一行的兩個子行在較小螢幕上如何處理？
   - 第二行的統計摘要在小螢幕上是否換行？

4. **統計摘要與客戶選擇控制之間是否需要分隔線？**
   - 類似 `[全選客戶] [一鍵展開] | [統計摘要]` 的視覺分隔？

請確認我的理解是否正確，特別是：
- ✅ 第一行確實分成兩個子行
- ✅ 統計摘要移到第二行，與客戶選擇控制同一行
- ✅ 操作按鈕（刷新、批量分配）放在第一行的第二子行
